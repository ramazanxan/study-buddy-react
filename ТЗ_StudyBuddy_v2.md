# StudyBuddy — Техническое задание

**Версия:** 2.0 | **Дата:** 06.06.2026 | **Тип:** Веб-приложение (адаптив: десктоп + мобайл)

---

## 1. Общие сведения

**Название:** StudyBuddy  
**Слоган:** «Найди своих по учёбе»

**Проблема.** В своей академической группе студент часто не находит единомышленников — людей «на одной волне» или с теми же увлечениями. А в соседних группах и на других факультетах они есть, но познакомиться негде.

**Цель.** Платформа знакомств для учёбы («Тиндер для университета»): студенты находят партнёров по целям и интересам, объединяются, мотивируют друг друга и не бросают начатое.

**Аудитория.** Студенты КГТУ им. И. Раззакова (Бишкек, KG), 17–25 лет.

**Платформа.** Веб-сайт, адаптивный под телефон.

**Язык интерфейса.** Русский с кыргызскими культурными акцентами (Агай/Эже, KG-код).

## 2. Концепция и ключевые отличия (УТП)

| Фича | Что делает | Зачем |
|---|---|---|
| **«Искра»** | Алгоритм мэтчит по совместимости целей, а не только по вузу | Находит людей, чьи цели дополняют твои |
| **«Я тоже хочу это»** | Под описанием «о себе» — кнопка; 5+ нажатий → авто-предложение создать группу | Само находит команду без поиска и фильтров |
| **«Позови меня»** | Статус «ищу здесь и сейчас»: партнёра на хакатон, курсовую и т.д. | Актуально сегодня, завтра можно сменить |
| **«Пакт»** | Двое ставят общую цель, она публична на их профилях | Социальное давление — бросить стыдно |
| **Еженедельный чек-ин** | Каждое воскресенье: «Что сделал за неделю?» | Возвращает на сайт каждую неделю |
| **Агай/Эже** | Менторство: старшекурсник берёт шефство над первокурсником | Культурный код KG, престижно |
| **Репутация / «Надёжный партнёр»** | Бейдж и место выше в ленте за выполненные цели | Стимул не бросать |
| **Доска побед** | Лента результатов с фото/текстом, видна всем | Мотивация + контент сам по себе |
| **Маркетплейс** | Продажа книг, ноутбуков и т.п. | Польза + удержание |

## 3. Роли пользователей

1. **Гость** — видит лендинг, может зарегистрироваться/войти.
2. **Студент** — основной пользователь: профиль, лента, лайки, чат, цели, чек-ин, маркетплейс.
3. **Наставник (Агай/Эже)** — студент 3–5 курса с бейджем наставника, ведёт подопечных.
4. **Модератор** — помощник администратора, модерирует контент в назначенных разделах.
5. **Администратор** — полное управление платформой: пользователи, контент, настройки, статистика.

## 5. Функциональные требования по экранам

### 5.1. Лендинг

- Hero: заголовок «Найди своих по учёбе», описание платформы, 3D-кнопка **«Регистрация»** → /register, ссылка «Войти».
- Reveal-анимации при скролле, плавное idle-движение плавающих фигур.
- Декоративный паттерн — кыргызские угловые орнаментальные мотивы (CSS repeating-gradient).
- Блоки: «Как работает» (3 шага), 6 ключевых фич, превью Доски побед (3 карточки), призыв к регистрации, футер.

### 5.2. Регистрация / Вход

**Вход:** логин + пароль.

**Регистрация — два шага на одной странице:**

**Шаг 1 — Личные данные:**
- Выбор роли вверху: **👨‍🎓 Студент** | **🧭 Наставник** (таб-переключатель)
  - При выборе «Наставник» — инфо-баннер: *«Наставник — студент 3–5 курса, который помогает младшим. После регистрации вы получите бейдж Агай/Эже.»*
- Логин — **только латиница** (`a–z`, `0–9`, `_`), 3–32 символа, уникальный
- Имя (обязательно)
- Возраст (14–80, обязательно)
- Пароль — **минимум 8 символов**, живая проверка: ✓ 8+ символов, ✓ Заглавная, ✓ Строчная, ✓ Цифра; индикатор силы пароля (слабый / средний / сильный / очень сильный)
- Повтор пароля

**Шаг 2 — Учёба:**
- Институт (dropdown, полный список институтов КГТУ — см. §8)
- Направление (зависит от института, обновляется динамически — см. §8)
- Курс (1–5, чипы-кнопки)
- «О себе» — textarea: *«Расскажите о себе — интересы, цели, чем занимаетесь...»* (необязательно)

**Подзаголовок формы:** «Заполните данные для регистрации»

**Доп. поля (дозаполнить в профиле позже):** ФИО полностью, группа, фото, увлечения/интересы.

**Валидация:** уникальность логина, совпадение паролей, корректность возраста, обязательные поля.

### 5.3. Профиль

- Просмотр своего и чужого профиля.
- Редактирование всех данных, загрузка/смена фото.
- Поля: фото (или инициалы-аватар), ФИО, логин (@login), институт, группа, курс, возраст, увлечения (теги), о себе, счётчик «Я тоже хочу это».
- **Бейджи:** Агай/Эже (наставник), «Надёжный партнёр», уровень репутации, активные Пакты.
- Статистика: репутация, кол-во мэтчей, выполненных целей.
- Кнопки на чужом профиле: ❤️ Лайк, 💬 Написать (если мэтч), 🙋 Я тоже хочу это.
- Кнопка «Стать наставником» (если курс ≥ 3).

### 5.4. Лента и мэтчинг

- Карточки студентов: фото/инициалы, имя, институт, курс, текст «О себе» (2 строки), теги интересов (до 3), репутация, метка ⚡ Искра.
- **Фильтры:** направление, институт, курс (чипы 1–5), возраст (от/до), поиск по имени, тогл «Только Искра».
- **«Искра»** — отметка на карточках с высоким баллом совместимости (≥ 6).
- **Лайк** → при взаимном = **Мэтч** с анимацией конфетти. Открывается чат.
- Если мэтч уже есть — кнопка «Написать» вместо лайка.
- Сортировка: по баллу Искры, затем по репутации.

### 5.5. Чат

- Список диалогов (мэтчи): аватар, имя, превью последнего сообщения, время, счётчик непрочитанных.
- Окно сообщений в стиле мессенджера: свои — справа (primary-цвет), чужие — слева (белая карточка).
- Поле ввода + кнопка отправки; статусы «прочитано».
- Прикрепление фото и файлов.
- Пустое состояние: «Нет совпадений. Лайкай студентов в ленте, чтобы начать общение!»

### 5.6. Цели, Чек-ин, Пакт, Позови меня

**Вкладка «Мои цели»:**
- Добавить цель (поле + кнопка).
- Карточки целей: текст, прогресс-бар (слайдер), статус (активна/выполнена/сорвана).
- Завершение цели → конфетти-анимация; срыв → мягкое приглушённое состояние.

**Вкладка «Пакты»:**
- Создать пакт: выбрать партнёра из мэтчей, ввести цель, дедлайн.
- Карточки: оба аватара, цель, дедлайн, статус (активен/выполнен/сорван) с цветовой кодировкой.
- Пакт публичен на профилях обоих участников.

**Вкладка «Чек-ин»:**
- Карточка текущей недели: «Что сделал за эту неделю?» — textarea + отправить.
- История прошлых чек-инов.
- Пропуск 2 недель → напоминание от партнёра.

**Вкладка «Позови меня»:**
- Создать/очистить свой статус-приглашение (текст: «ищу партнёра на хакатон» и т.д.).
- Список статусов других пользователей с кнопкой «Откликнуться».

### 5.7. Менторство (Агай/Эже)

**Для курса ≥ 3 (вид наставника):**
- Включить/выключить режим наставника.
- Список входящих запросов (принять/отклонить).
- Список активных подопечных.

**Для курса 1–2 (вид студента):**
- Список доступных наставников (карточки с именем, институтом, курсом).
- Кнопка «Запросить наставника».
- Отображение текущего наставника (если есть).

### 5.8. Доска побед

- Кнопка «Поделиться победой» → форма: текст + необязательное фото.
- Сетка карточек: аватары авторов, текст победы, фото (если есть), дата, ❤️ реакции.
- Фильтр: Все / Мои победы.

### 5.9. Маркетплейс

- Фильтр по категориям: Все / Книги / Электроника / Одежда / Другое.
- Карточки: фото/заглушка, название, цена (сом), категория, продавец.
- Добавить объявление: название, цена, категория, описание.
- «Написать продавцу» — если мэтч → чат; иначе тост «Сначала совпадите!»

### 5.10. Админ-панель

Доступна только роли `admin`. Макет: боковое меню (sidebar) + основная область. На мобайле — горизонтальный таб-бар.

#### 📊 Дашборд

Карточки-статистика (6 штук):
- 👨‍🎓 Всего студентов
- 🆕 Новых за неделю
- 🤝 Активных менторств
- ⚠️ Жалоб
- 👥 Учебных групп
- 📅 Ближайших мероприятий

Два виджета:
- «Последние регистрации» — список 5 последних пользователей (аватар, имя, институт, дата)
- «Ближайшие мероприятия» — 3 события с датой, названием, местом

#### 👨‍🎓 Студенты

- Поиск по ФИО, логину, группе, курсу
- Фильтры: институт (dropdown), курс (чипы 1–5), статус (Все / Активные / Заблокированные)
- Таблица: аватар, ФИО, логин, институт, курс, группа, репутация, статус
- Статусы: Активен (зелёный), Заблокирован (красный), Наставник (фиолетовый)
- Действия: 👁 Профиль, 🚫 Заблокировать / ✅ Разблокировать, 🏅 +/− Бейдж «Надёжный партнёр»
- Пагинация: 10 строк на страницу

#### 🧭 Наставники

- Таблица наставников (isMentor = true)
- Колонки: аватар, ФИО, логин, институт, курс, кол-во подопечных, статус
- Действия: профиль, заблокировать/разблокировать

#### 🛡️ Модераторы

- Карточки модераторов: имя, назначенные разделы, последняя активность
- Кнопка «Добавить модератора»

#### 📰 Новости

- Форма: заголовок + текст → «Опубликовать»
- Список новостей: заголовок, текст (превью), дата
- Действия: 📌 Закрепить/Открепить, 🗑 Удалить
- Закреплённые новости отмечены значком и выделены рамкой

#### 📢 Объявления

- Форма: текст + важность (Обычное / Важное / Срочное) + срок действия
- Список с цветовой кодировкой: обычное — синее, важное — оранжевое, срочное — красное
- Действия: 📌 Закрепить, 🗑 Удалить

#### 🔔 Уведомления

- Выбор получателей: Все студенты / По институту / Новые пользователи
- Textarea сообщения + кнопка «Отправить»
- Тост: «Уведомление отправлено X пользователям»
- История отправленных уведомлений (последние 5)

#### 📁 Файлы

- Зона загрузки (drag-and-drop UI) + кнопка «Загрузить файл»
- Список файлов: иконка типа, название, размер, дата, «Скачать», «Удалить»

#### 📈 Статистика

- Сводные карточки: всего студентов, всего мэтчей, входов за неделю
- Горизонтальный бар-чарт: студентов по институтам
- Недельный столбчатый чарт: активность по дням (Пн–Вс)
- Топ-5 студентов по репутации (ранжированный список с аватарами)
- Мэтчи за месяц с трендом

#### ⚙️ Настройки

- Название платформы (input)
- Слоган (input)
- Макс. подопечных у наставника (number)
- Регистрация открыта (toggle)
- Подтверждение аккаунтов вручную (toggle)
- Кнопка «Создать резервную копию»
- Кнопка «Сохранить настройки»

## 8. Ключевые алгоритмы

### «Искра» (совместимость)

```
score = (общие интересы × 3) + (дополняющие цели × 5) + (один институт × 2)

Дополняющие пары целей/интересов:
  «стартап» ↔ «сотрудник»
  «программирование» ↔ «разработка»
  «дизайн» ↔ «ui» / «ux»
  «исследование» ↔ «наука»
  «бизнес» ↔ «предприниматель»
  «медицин» ↔ «здоровь»
  «право» ↔ «юрист»
  «запустить» ↔ «участвовать»
  «учить» ↔ «изучать»

Порог Искры: score ≥ 6
```

### Авто-группа

При 5+ нажатиях «Я тоже хочу это» на одном профиле → всплывающее предложение создать учебную группу.

### Репутация

- +очки за выполненную цель
- +очки за сдержанный пакт
- −очки за пропуск чек-инов (2+ недели подряд)
- Порог репутации → бейдж «Надёжный партнёр» + повышение в ленте

### Напоминания

- Cron каждое воскресенье → уведомление «Время чек-ина!»
- При пропуске 2 недель → уведомление от партнёра по пакту

## 11. Технический стек

| Слой | Выбор | Обоснование |
|---|---|---|
| **Frontend** | React 19 + Vite 8, JSX | Быстрый HMR, современный стек |
| **Роутинг** | React Router DOM v6 | Стандарт для React SPA |
| **Стейт (MVP)** | React Context + localStorage | Нет бэкенда на старте |
| **Стили** | CSS-переменные, без Tailwind | Полный контроль, кастомный дизайн |
| **Backend (prod)** | Supabase | Auth + БД + Storage + Realtime из коробки |
| **БД** | PostgreSQL 15+ | Реляционные связи, JOIN-ы, GIN-индексы |
| **Realtime-чат** | Supabase Realtime | WebSocket-подписки на таблицу messages |
| **Файлы** | Supabase Storage | Бакеты: avatars, chat-attachments, listings, wins |
| **Auth** | Supabase Auth (JWT) | Пароли только хэшем (bcrypt/argon2) |
| **Хостинг** | Frontend → Vercel; БД → Supabase | Бесплатный старт |

### Безопасность

- Логин: `^[a-zA-Z0-9_]{3,32}$`
- Пароль: 8+ символов, заглавная + строчная + цифра (проверка live на фронте и на бэке)
- Row Level Security (Supabase): пользователь видит/меняет только свои данные; чат — только участники
- Защита от спама: rate limiting на лайки и сообщения
- Пароли: только хэш, никогда plain-text
- Admin-эндпоинты: проверка роли на бэкенде

## 12. База данных (PostgreSQL DDL — основные таблицы)

```sql
CREATE TYPE user_role     AS ENUM ('student', 'mentor', 'admin');
CREATE TYPE goal_status   AS ENUM ('active', 'done', 'failed');
CREATE TYPE pact_status   AS ENUM ('active', 'done', 'failed');
CREATE TYPE listing_state AS ENUM ('active', 'sold', 'hidden');

CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  login         VARCHAR(32) UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  full_name     VARCHAR(120),
  faculty       VARCHAR(120),
  direction     VARCHAR(120),
  group_name    VARCHAR(60),
  course        SMALLINT CHECK (course BETWEEN 1 AND 5),
  age           SMALLINT CHECK (age BETWEEN 14 AND 80),
  photo_url     TEXT,
  about         TEXT,
  role          user_role NOT NULL DEFAULT 'student',
  reputation    INTEGER NOT NULL DEFAULT 0,
  is_mentor     BOOLEAN NOT NULL DEFAULT FALSE,
  is_banned     BOOLEAN NOT NULL DEFAULT FALSE,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE user_interests (
  user_id     UUID REFERENCES users(id) ON DELETE CASCADE,
  interest    VARCHAR(40) NOT NULL,
  PRIMARY KEY (user_id, interest)
);

CREATE TABLE badges (
  code VARCHAR(40) PRIMARY KEY,
  name VARCHAR(80) NOT NULL
);
CREATE TABLE user_badges (
  user_id    UUID REFERENCES users(id) ON DELETE CASCADE,
  badge_code VARCHAR(40) REFERENCES badges(code),
  granted_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  PRIMARY KEY (user_id, badge_code)
);

CREATE TABLE likes (
  from_user  UUID REFERENCES users(id) ON DELETE CASCADE,
  to_user    UUID REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  PRIMARY KEY (from_user, to_user),
  CHECK (from_user <> to_user)
);

CREATE TABLE matches (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_a     UUID REFERENCES users(id) ON DELETE CASCADE,
  user_b     UUID REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (user_a, user_b)
);

CREATE TABLE chats (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  match_id   UUID REFERENCES matches(id) ON DELETE CASCADE,
  is_group   BOOLEAN NOT NULL DEFAULT FALSE,
  title      VARCHAR(120),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE TABLE messages (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  chat_id     UUID REFERENCES chats(id) ON DELETE CASCADE,
  sender_id   UUID REFERENCES users(id) ON DELETE SET NULL,
  body        TEXT,
  attachments JSONB DEFAULT '[]',
  is_read     BOOLEAN NOT NULL DEFAULT FALSE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE goals (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  owner_id   UUID REFERENCES users(id) ON DELETE CASCADE,
  text       TEXT NOT NULL,
  status     goal_status NOT NULL DEFAULT 'active',
  progress   SMALLINT NOT NULL DEFAULT 0 CHECK (progress BETWEEN 0 AND 100),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE TABLE goal_wants (
  goal_id    UUID REFERENCES goals(id) ON DELETE CASCADE,
  user_id    UUID REFERENCES users(id) ON DELETE CASCADE,
  PRIMARY KEY (goal_id, user_id)
);

CREATE TABLE pacts (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_a     UUID REFERENCES users(id) ON DELETE CASCADE,
  user_b     UUID REFERENCES users(id) ON DELETE CASCADE,
  goal_text  TEXT NOT NULL,
  deadline   DATE,
  status     pact_status NOT NULL DEFAULT 'active',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE checkins (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID REFERENCES users(id) ON DELETE CASCADE,
  goal_id    UUID REFERENCES goals(id) ON DELETE SET NULL,
  week_start DATE NOT NULL,
  text       TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (user_id, week_start)
);

CREATE TABLE callme (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID REFERENCES users(id) ON DELETE CASCADE,
  text       TEXT NOT NULL,
  expires_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE mentorships (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  mentor_id  UUID REFERENCES users(id) ON DELETE CASCADE,
  mentee_id  UUID REFERENCES users(id) ON DELETE CASCADE,
  status     VARCHAR(20) NOT NULL DEFAULT 'pending',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (mentor_id, mentee_id)
);

CREATE TABLE wins (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  author_id  UUID REFERENCES users(id) ON DELETE CASCADE,
  partner_id UUID REFERENCES users(id) ON DELETE SET NULL,
  text       TEXT,
  photo_url  TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE listings (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  seller_id   UUID REFERENCES users(id) ON DELETE CASCADE,
  title       VARCHAR(160) NOT NULL,
  price       NUMERIC(10,2) NOT NULL CHECK (price >= 0),
  category    VARCHAR(60),
  description TEXT,
  photos      JSONB DEFAULT '[]',
  state       listing_state NOT NULL DEFAULT 'active',
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```
